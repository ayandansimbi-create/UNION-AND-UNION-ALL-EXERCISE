UNION AND UNION ALL EXERCISE

In this exercise, I used the Databricks SQL environment to write and execute queries focused on the UNION and UNION ALL operators.

Using the datasets provided in the question paper, I created CSV files and uploaded them into Databricks as part of a data ingestion process. The exercise consisted of multiple banking-related datasets covering various business scenarios, including ATM transactions, customer complaints, loan applications, and other financial records.

Throughout the exercise, I applied the UNION operator to combine data from related tables while removing duplicate records. I also used the UNION ALL operator when it was necessary to retain every row from the source tables, including duplicates, to support accurate reporting and transaction analysis.

This project strengthened my understanding of set operations in SQL, data ingestion workflows in Databricks, and the practical use of UNION and UNION ALL in real-world banking and financial data scenarios.
